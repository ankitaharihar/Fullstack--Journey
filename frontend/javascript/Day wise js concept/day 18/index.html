Day 18

Async Await, Request Methods and API Practice

Aaj ka topic asynchronous JavaScript ka core part hai. Niche concept summary aur live API fetch demo dono diye gaye hain.

1) Async / Await

Async function Promise return karta hai. Await se Promise resolve hone ka wait karte hain, jisse code readable ho jata hai. 

async function getRecipe() {
  try {
    const response = await fetch("https://dummyjson.com/recipes/1");
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}

2) HTTP Request Methods

  • GET: Data retrieve karne ke liye.
  • POST: Naya data create karne ke liye.
  • PUT: Existing data update karne ke liye.
  • DELETE: Data remove karne ke liye.

3) Try / Catch

Network call fail hone par app crash na ho, isliye risky async code ko try block me rakhte hain aur error ko catch block me handle karte hain. 

try {
  const data = await apiCall();
} catch (error) {
  console.error("Request failed", error);
}

4) Request Data Parts

  • Query Params: URL ke end me jaate hain, jaise ?id=1 ya ?limit=10&prefix=ch.
  • Body: POST/PUT requests me actual data payload.
  • Headers: Metadata, jaise content type ya token.
