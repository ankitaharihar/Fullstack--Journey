Variables in javascript

A variable is a container that stores data which can be used later in a program.

var, let, const

  • var - function scoped, can be redeclared and updated, hoisted with undefined
  • let - block scoped, cannot be redeclared but can be updated, hoisted with temporal dead zone
  • const - block scoped, cannot be redeclared or updated, hoisted with temporal dead zone

rules in JavaScript

  • Variable names must start with a letter, underscore, or dollar sign
  • Variable names can contain letters, numbers, underscores, and dollar signs
  • Variable names cannot be reserved keywords (e.g., var, let, const, if, else, etc.)
  • Variable names are case-sensitive (e.g., name and Name are different variables)
  • Variable names should be descriptive and meaningful to improve code readability
  • camelCase is the preferred naming convention in JavaScript

Data types in JavaScript

Primitive data -

Primitive data types are immutable and stored directly in the variable.

  • Primitive data types - string("",'',``- string template, string manipulation ), bigint,number, boolean, null, undefined, symbol

    • Non-primitive data -

    Non-primitive data types are mutable and stored as references to the actual data.

  • Non-primitive data types - object, array, function

Operators in JavaScript

  • Arithmetic operators - +, -, *, /, %, ++, --
  • Assignment operators - =, +=, -=, *=, /=, %=
  • Comparison operators - ==, ===, !=, !==, >, <, >=, <=
  • Logical operators - &&, ||, !
  • Bitwise operators - &, |, ^, ~, <<, >>, >>>
  • Ternary operator - condition ? expression1 : expression2

Control structures in JavaScript

  • Conditional statements - if, else if, else, switch
  • Looping statements - for, while, do while, for...in, for...of
  • Jump statements - break, continue, return

Functions in JavaScript

  • Function declaration - function name(parameters) { // function body }
  • Function expression - const name = function(parameters) { // function body }
  • Arrow function - const name = (parameters) => { // function body }
  • Higher-order functions - functions that take other functions as arguments or return functions

Objects in JavaScript

  • Object literal - const obj = { key: value }
  • Constructor function - function Person(name, age) { this.name = name; this.age = age }
  • Class syntax - class Person { constructor(name, age) { this.name = name; this.age = age } }
  • Prototype - objects can inherit properties and methods from other objects through the prototype chain

Arrays in JavaScript

  • Array literal - const arr = [1, 2, 3]
  • Array constructor - const arr = new Array(1, 2, 3)
  • Array methods - push, pop, shift, unshift, splice, slice, forEach, map, filter, reduce

Scope in JavaScript

  • Global scope - variables declared outside of any function or block
  • Function scope - variables declared within a function are only accessible within that function
  • Block scope - variables declared with let or const within a block (e.g., if statement, for loop) are only accessible within that block

Hoisting in JavaScript

Hoisting is a JavaScript mechanism where variables and function declarations are moved to the top of their containing scope during the compilation phase. This means that you can use variables and functions before they are declared in the code.

  • Variable hoisting - variables declared with var are hoisted and initialized with undefined, while variables declared with let and const are hoisted but not initialized (temporal dead zone)
  • Function hoisting - function declarations are hoisted and can be called before they are defined, while function expressions and arrow functions are not hoisted
