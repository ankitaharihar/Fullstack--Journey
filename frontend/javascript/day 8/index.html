user defined functions

Functions which are defined by user, for example:

   function functionName(parameters) {
        // function body
    }

variale number of arguments (....nums)

We cant declare same scope variables in same function
return gives the value of the function, return always with condition

Hoisting - it means that variable and function declarations are moved to the top of their scope before code execution

global scope - we access global variables in local scope

ex let num = 10; // global variable function example() { console.log(num); // accessing global variable in local scope }

local scope - we access local variables in global scope

function example() { let num = 10; // local variable }

Shadowing means - when a local variable has the same name as a global variable, the local variable shadows the global variable within its scope

tdz - temporal dead zone means let and const variables are not accessible before their declaration
